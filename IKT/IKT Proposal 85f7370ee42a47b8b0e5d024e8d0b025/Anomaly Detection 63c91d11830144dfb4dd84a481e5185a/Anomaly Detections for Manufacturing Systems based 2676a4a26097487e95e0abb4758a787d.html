Anomaly Detections for Manufacturing Systems based on Sensor Data

Source

Abstract

When following a fine-grained monitoring of manufacturing systems (machines), two challenges emerge:

  1. The feeding procedure of large amounts of sensor data to downstream processing components.
  1. The meaningful analysis of the produced data.

Introduction

  • Predictive Maintenance: selection of appropriate technique from the field of PM is challenging
  • Other approaches: Condition Monitoring or Continuous Improvements.
  • Trend towars ML: can machine failures be predicted??? → practical application of ML algorithms is still challenging

The paper provides two examples:

  1. Pharma packaging system → continuous monitoring process generates large amounts of data, machines may be individualized for each customer
  1. 3D printing machines → slower process as above, but also measurements of the environment, e.g. room temperature

Main objective of Anomaly Detection (AD) is to analyze streaming data to develop models that can be used with an appropriate number of parameters and across applications. The selection of the used algorithm as well as their parametrization is still challenging (no standard software available).

Related Work

Acquisition of Data

  • OPC UA for data transmission and PTP (precision time protocol) for time synchronization [faltinski]
  • Representation and transformation of sensor data in cyber-physical systems [jirkovsky]

AD techniques

  • statistical methods, e.g. statistical profiling, parametric statistical modeling
  • machine learning approaches

Discussion

Concerning the acquisition of sensor data, different sensor systems exist. The use different data models and provide different connection and communication methods → common exchange model is necessary.

OPC UA (Open Platform Communications Unified Architecture): widely used in industry to transfer data between systems and to call PLC-related functions. For the cases 1 in the paper, OPC UA was not sufficient due to bigger message sizes (overall dataset contains of ~22GB). → OPC UA is not designed to continuously transfer large amounts of data.

After acquisition of sensor data, it has to be

  • pre-processed to convert it into analysis-compliant data models
  • reduced in its noise
  • normalized for further analysis
  • windowed (for stream-enabled sensor data) by creating data windows depending on the machine cycle speed (kappa or lambda architecture)

The selection of proper anomaly detection algorithm is a challenging task → goal is a parameterless algorithm to support multiple machine variants.
